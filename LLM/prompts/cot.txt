# 皮肉検出タスク
ユーザーの発言とそれに対する応答を分析し、応答が皮肉かどうかをステップバイステップで判断してください。

## 出力形式
出力は、必ず以下のJSON構造に厳密に従ってください。  
**他のテキストや説明は一切含めないでください。**

```json
{
  ""analysis"": [
    ""1. ユーザーの発言の分析: ..."",
    ""2. 応答の表面的な意味の分析: ..."",
    ""3. 応答の真意の推測: ..."",
    ""4. 表面的意味と真意の乖離の確認: ..."",
    ""5. 最終判断の理由: ...""
  ],
  ""label"": ""IRONY"" // または ""NOT_IRONY""
}
```

## 分析手順
analysis配列には、以下の手順に沿った思考過程を記述してください。
1.  ユーザーの発言の分析
2.  応答の表面的な意味の分析
3.  応答の真意の推測
4.  表面的意味と真意の乖離の確認
5.  最終判断の理由

---

## 例

**例1:**
-   **ユーザーの発言:** このレストラン、すごく評価が高いんだって！
-   **応答:** うん、この生温かいスープとゴムみたいなステーキは、まさに至高の味だね。

```json
{
  ""analysis"": [
    ""1. ユーザーの発言の分析: ユーザーは、レストランの高い評価を信じ、料理に対して肯定的な期待感を表明している。"",
    ""2. 応答の表面的な意味の分析: 「至高の味」という言葉を使い、料理を最高級だと褒めている。"",
    ""3. 応答の真意の推測: 「生温かいスープ」「ゴムみたいなステーキ」という具体的な描写は、明らかに料理が不味いことを示唆している。話し手は、期待外れの料理を食べてがっかりしている。"",
    ""4. 表面的意味と真意の乖離の確認: 表面上の「最高の賛辞」と、実際の「最低の評価」との間には極めて大きな乖離がある。"",
    ""5. 最終判断の理由: 否定的な具体例を挙げつつ、あえて逆の肯定的な言葉（至高の味）で締めくくることで、レストランの評価やユーザーの期待を揶揄する、典型的な皮肉表現となっている。""
  ],
  ""label"": ""IRONY""
}
```

**例2:**
-   **ユーザーの発言:** 今日のプレゼンどうだった？
-   **応答:** すごく分かりやすかったよ！特にデータの説明が丁寧で助かった。

```json
{
  ""analysis"": [
    ""1. ユーザーの発言の分析: ユーザーは、自身のプレゼンテーションに対する率直なフィードバックを求めている。"",
    ""2. 応答の表面的な意味の分析: プレゼンが「すごく分かりやすかった」と全体を肯定し、「データの説明が丁寧で助かった」と具体的な良い点を挙げている。"",
    ""3. 応答の真意の推測: 表現は一貫して肯定的であり、具体的な褒め言葉も含まれているため、隠された否定的な意図があるとは考えにくい。心からの称賛と推測される。"",
    ""4. 表面的意味と真意の乖離の確認: 表面的な意味と真意は一致しており、乖離は見られない。"",
    ""5. 最終判断の理由: 肯定的な評価と具体的な理由が一致しており、言葉を裏返すような皮肉の要素は含まれていない。""
  ],
  ""label"": ""NOT_IRONY""
}
```
